# Will Kelly custom GPT content pipeline JSONs

This package contains an interview-ready custom GPT-driven content pipeline for technical and marketing content.

## What it demonstrates

- AI as a governed workflow system, not a drafting shortcut
- Structured intake before drafting
- Routing between technical, marketing, hybrid, and enablement workflows
- Quality gates for accuracy, claims, clarity, and publication readiness
- Repurposing only after source content is approved
- A practical way to discuss AI enablement in job interviews

## How to use this in an interview

Lead with the orchestrator. It shows systems thinking.

Then show the intake GPT. It proves you know bad content starts before drafting.

Then show the QA governance GPT. That is the trust layer.

Close with the repurposing GPT. That shows how content becomes reusable infrastructure instead of one-off output.
